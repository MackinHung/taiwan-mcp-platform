信任與安全

用最白話的方式，告訴你我們怎麼保護你的資料

30 秒讀懂重點

  • 1. 我們不存你的密碼 — 登入是透過 GitHub 或 Google 處理的，平台完全不經手。
  • 2. 平台是純轉接站 — 你查的資料直接來自政府 API，我們只是幫你轉送，不會偷看或修改。
  • 3. 每個工具都有營養標示 — 就像食品包裝一樣，讓你一眼看出安全等級。
  • 4. 上架前先掃描再審核 — 所有工具都經過自動安全掃描 + 人工審查，才能出現在市集裡。
  • 5. 有人搗蛋我們會擋 — 系統 24 小時監控異常行為，自動阻擋可疑活動。

平台怎麼運作？

簡單說，我們是一個安全的中間人。你問問題，我們把問題轉給台灣政府的公開資料庫，拿到答案後原封不動地還給你。整個過程中，我們不儲存你查詢的內容，也不會修改政府的原始資料。

你 → MCP 平台（安全轉接）→ 政府資料庫（原始資料）→ 結果回傳給你
技術細節：唯讀代理架構

平台採用唯讀代理（Read-only Proxy）架構。所有 API 請求經由平台轉發至政府端點，回應直接回傳給使用者。平台不持久化查詢內容或回應資料，僅記錄匿名使用量統計。所有通訊透過 HTTPS/TLS 1.3 加密。

六大安全保護

以下是我們為你做的六件事，確保你可以安心使用。

登入不經手密碼

你用 GitHub 或 Google 帳號登入，密碼由它們保管，平台完全碰不到。 就像你用 Google 登入其他網站一樣，只授權基本資訊，不用另外記密碼。

技術細節

OAuth 2.0 + PKCE 授權碼流程，防止授權碼被攔截。Session 以 CSRF Token 保護，Cookie 設定 Secure + HttpOnly + SameSite=Lax，防止跨站腳本（XSS）竊取。

金鑰不存明碼

你的 API 金鑰經過單向加密後才儲存 — 就像把文件碎掉再存， 即使有人偷看資料庫，也無法還原你的金鑰。另外，我們限制每分鐘的使用次數， 防止有人大量濫用。

技術細節

API Key 以 SHA-256 雜湊儲存（不可逆）。Rate Limiting 分級：匿名 30 req/min、認證用戶 50-2000 req/min（依方案）。所有通訊強制 HTTPS 加密。

伺服器有人看管

平台跑在 Cloudflare 的全球網路上，有專業的防火牆和防攻擊系統 24 小時守護。 不像傳統伺服器有被入侵的風險 — 我們的架構從設計上就沒有可以「闖入」的門。

技術細節

Cloudflare Workers 無伺服器架構（Serverless），無 SSH 攻擊面。Cloudflare WAF（Web Application Firewall）+ 自動 DDoS 防護。基礎設施通過 ISO 27001、SOC 2 Type II 國際安全認證。

每個工具都有營養標示

就像食品包裝上有營養成分表，每個 MCP 工具也有自己的安全成分表， 包含四個維度：程式碼是否公開、會碰到什麼資料、有多少權限、多少人在用。 你不需要懂技術，看顏色就能判斷。

開源已審計 公開資料 唯讀 信賴

綠色 = 安心  ·  藍色 = 良好  ·  橘色 = 留意  ·  紅色 = 小心

可疑行為自動攔截

如果有人短時間內瘋狂呼叫 API、從陌生國家突然登入、或一直嘗試輸入錯誤密碼， 系統會立刻發現並自動攔截，同時通知管理員處理。

技術細節

即時監控三種異常模式：API Key 速率突增（Rate Spike）、Session 地理跳躍（Geo Anomaly）、認證失敗連續累積（Brute Force Detection）。觸發時自動記錄 + 管理員告警。

符合台灣法規

我們遵守台灣個資法，你的資料受法律保障。 如果不幸發生資料外洩（雖然可能性極低），我們承諾 72 小時內通報， 而且有完整的緊急處理程序。

技術細節

台灣個人資料保護法（PDPA）合規。APEC CBPR（亞太經合組織跨境隱私規則）保障跨境資料。72 小時資料外洩通報承諾。完整事件回應 SOP（三級分類 L1/L2/L3，含容器化、通知、復原流程）。

A～D 信任等級 — 怎麼看？

每個工具卡片的右上角有一個 A、B、C、D 等級，就像學校成績一樣 — A 最好，D 要小心。 這個等級是系統根據四個面向自動算出來的，開發者無法自己改。

A

放心用

14 ～ 16 分

程式碼公開透明、只讀公開資料、不會代你做任何修改、已有大量人使用。可以安心使用。

B

不錯

11 ～ 13 分

大部分面向表現良好。可能需要你的帳號資料、或能做有限的寫入操作。日常使用沒問題。

C

看清楚再用

8 ～ 10 分

可能會碰到你的個人資料、或需要較大的操作權限。建議先看一下它的安全聲明和標章。

D

要特別小心

4 ～ 7 分

程式碼沒有公開、可能碰到敏感資料、或需要很高的權限。除非你很信任這個來源，否則建議觀望。
