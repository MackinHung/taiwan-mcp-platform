# 資料外洩事件處理 SOP

> **版本**: 1.0
> **最後更新**: 2026-03-18
> **負責**: 台灣 MCP 平台維運團隊

---

## 平台風險評估

本平台為**唯讀代理架構**，僅代理台灣政府公開資料，不儲存敏感個人資訊。

### 需要保護的資產（僅三項）

| 資產 | 被盜用的後果 | 防護措施 |
|------|------------|---------|
| API Keys | 冒用呼叫額度 | SHA-256 hash 儲存、KV cache |
| OAuth Sessions | 冒充身份 | Secure + HttpOnly Cookie、CSRF Token |
| Email | 垃圾郵件風險（量少） | OAuth 委託、不存密碼 |

### 攻擊面分析

| 攻擊向量 | 一般平台風險 | 本平台風險 | 原因 |
|---------|------------|----------|------|
| SQL Injection 竊資 | 高 | 極低 | 無敏感資料可偷 |
| 資料庫被 dump | 高 | 低 | D1 僅存 API key hash + email |
| 惡意檔案上傳 | 高 | 無 | 不接受用戶上傳執行檔 |
| 資料篡改 | 高 | 無 | 不寫入政府資料 |
| API Key 被盜 | 中 | 中 | 唯一有意義的攻擊面 |
| OAuth Session 劫持 | 中 | 低 | Secure cookie + CSRF 已防護 |

---

## 事件分級

| 等級 | 定義 | 範例 |
|------|------|------|
| **L1 低** | 非個資相關的安全事件 | DDoS、服務中斷、異常流量 |
| **L2 中** | 少量 API Key 洩露 | 單一用戶 key 外洩、session 異常 |
| **L3 高** | 大量用戶資料外洩 | D1 資料庫被存取、OAuth token 批量外洩 |

## 處理流程

### 1. 發現（T+0）

**觸發來源**：
- Cloudflare WAF 告警
- 異常偵測 middleware 記錄（`anomaly:{date}` KV 日誌）
- 用戶回報
- 人工發現

**立即行動**：
- 記錄：發現時間、發現方式、初步影響範圍
- 指定事件處理負責人
- 在內部頻道通報

### 2. 評估（T+0 ~ T+2h）

- 確認洩露範圍（哪些資料、多少用戶）
- 判斷事件等級（L1 / L2 / L3）
- **L1**：內部處理即可，記錄後歸檔
- **L2+**：進入遏止 + 通報流程

### 3. 遏止（T+2h ~ T+6h）

| 行動 | 適用等級 | 執行方式 |
|------|---------|---------|
| 撤銷受影響的 API Keys | L2+ | D1 `DELETE` + KV cache `delete` |
| 強制登出受影響的 Sessions | L2+ | D1 `DELETE` + KV `SESSION_CACHE` |
| 封鎖攻擊來源 IP | L2+ | Cloudflare WAF 規則 |
| 暫停 Gateway | L3 | Cloudflare Workers 下架（最後手段） |

### 4. 通報（T+6h ~ T+72h）

| 行動 | 適用等級 | 方式 |
|------|---------|------|
| 通知受影響用戶 | L2+ | Email 通知（含事件說明 + 建議行動） |
| 通報個資保護委員會 (PDPC) | **L3** | 72 小時內，依法定程序通報 |
| 公開聲明 | L3 | 平台公告頁發布事件聲明 |

### 5. 復原（T+72h+）

- 修補漏洞（root cause fix）
- 重新發行受影響的 API Keys
- 強制所有用戶重新登入（L3 時）
- 發布事件報告（含時間軸、影響範圍、改善措施）
- 更新安全措施
- 進行內部檢討會議
