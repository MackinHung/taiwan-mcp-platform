# 競品分析 → 建議方案

> **日期**: 2026-03-17
> **性質**: WG-3 建議方案
> **對象**: WG-1 + WG-2 + WG-4

---

## 給 WG-1（MCP 工廠）

### 立即行動
1. **Batch 2 開發** — 14 個新 server 已有完整規格（`wg-1-batch2-spec.md`）
2. **法律類優先** — taiwan-law / judgment / legislative 企業需求最高
3. **生活類次之** — taiwan-oil-price / reservoir / disaster 大眾感知度最高

### 下一步研究
1. **組合 MCP Server** — 跨 API 智慧組合（例：生活事件助手、智慧通勤）
2. **taiwan-housing 可行性** — 實價登錄無 REST API，需研究 CSV 批次方案
3. **taiwan-crypto** — MAX/BitoPro exchange API，需調查 API 穩定性

---

## 給 WG-2（平台迭代）

### 核心差異化功能（持續強化）
1. **營養標示** — 全球獨有，加入更多評估維度（回應速度、資料新鮮度）
2. **Composer UX** — 組合器是商業模式核心，需要更好的 UI
3. **Usage Dashboard** — server 使用量統計，增值功能

### 可參考但不模仿
- MCPize 的 85% 分潤模式 → 先觀察生態是否成熟
- Smithery 的 one-click hosting → 我們已有 Workers 基礎
- Glama 的大量收錄 → 我們走精品路線（品質 > 數量）

---

## 給 WG-4（安全）

### 持續強化
1. **營養標示是護城河** — 競品沒有信任評分，我們要持續領先
2. **考慮新維度**: API 回應速度、資料更新頻率、可用性 SLA
3. **合規研究**: 台灣個資法對 MCP server 的影響（WG-3 待研究）
